Open-source integration roadmap.

We are ready to start the planning and evaluation layer for open-source tools. We should not connect heavy engines directly until each workflow has stable save/load data contracts.

Are we ready?

Yes for roadmap, GitHub repo intake, license checks, and architecture planning. Not yet for full engine wiring until the related Morphic records are stable.

Safe integration roadmap

1. Save/load contracts

Define exactly what panels, assets, voices, cues, and rigs save to PostgreSQL.

2. Repo evaluation

Check license, hardware, cloud needs, API shape, security, and output files.

3. Adapter module

Build a replaceable service layer so tools can change without breaking Morphic.
