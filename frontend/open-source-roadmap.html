Open-source integration roadmap.

Yes — Morphic should be open-source-ready by design. Phase 1 locks down save/load data contracts and a transparent integration plan. Phase 2 wires AI enhancement and replaceable open-source adapters only after the core creator pipeline is stable.

looks_one Phase 1 — Do now

Open-source-ready website foundation

Build the website around data ownership first. The creator should be able to save a project, script, scenes, panels, characters, worlds, assets, workflow stages, and provider settings before any heavy engine is attached.

  • databaseSave/load contracts: every screen reads and writes Morphic records, not static mock data.
  • fact_checkLicense intake: every open-source tool must pass license, security, hardware, and output-format checks before use.
  • account_treeAdapter architecture: define a small service boundary so ComfyUI, Fabric.js, Konva.js, PixiJS, Redis/BullMQ, Socket.IO, OpenTimelineIO, and OpenColorIO can be swapped or postponed later.
  • inventory_2Asset Library first: every generated file must come back as an asset/storage object before export buttons are enabled.
looks_two Phase 2 — Next

AI enhancement + open-source adapters

After Phase 1 data is reliable, connect the AI-enhanced intake flow and start with thin adapters that create test assets instead of running every heavy workflow at once.

ComfyUIPrimary visual generation for characters, backgrounds, style workflows, comics, and animation assets.
Canvas stackFabric.js, Konva.js, and PixiJS for editing, layers, GPU previews, and rich editor tools.
Runtime stackNext.js, React, Node.js, Redis, BullMQ, and Socket.IO for app, queues, and real-time progress.
Research additionsOpenTimelineIO and OpenColorIO stay on the research list until timeline/color workflows need them.

Are we ready?

Yes for roadmap, GitHub repo intake, license checks, and architecture planning. Not yet for full engine wiring until the related Morphic records are stable.
