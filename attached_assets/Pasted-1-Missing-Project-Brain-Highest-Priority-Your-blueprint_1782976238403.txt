6. OpenRouter

I noticed something important in your backend.

You mentioned switching to OpenRouter.

However, the code currently calls:

https://api.openai.com/v1/chat/completions

So the backend doesn't yet match your intended AI provider abstraction.

I recommend introducing an AI Gateway so you can switch providers without changing business logic.

7. Backend responsibilities

The server currently mixes:

API routes
AI calls
Database logic

As the project grows, split it into:

controllers/

services/

agents/

database/

middleware/

utils/

This will make the codebase much easier to maintain.
